# Systems Under Siege - Reference Library

## What This Document Is Now

This file used to be the idea generator. It contained 8 "axes" that the assistant combined to produce post concepts (Method × Zone × Chaos Source × Trigger × Format × Time × Reader Stage × Product Category).

That approach is retired. Posts are now generated from **moments**, not from filling cells in a matrix (see SUSMaster.md for the new workflow).

This document survives as a **reference library** — lookup tables and lists for use *during* post generation, not as the source of ideas. When generating a post idea moment-first, consult this file:

- To check whether a method × zone combination has been overused
- To find the right zone name for tagging
- To remind yourself which methods exist when one of them is the right fit for the moment
- To spot the chaos sources or trigger types you've been leaning on too hard

When the assistant has a moment in hand and wants to know "what method does this look like?" or "what's the canonical zone name for this?" — this is the file.

## Reference 5: Method × Zone Matrix

This used to be the heart of the file. Now it's a *coverage check* — useful for spotting accidental repetition, not for filling empty cells.

The full current matrix lives in SUSIdeaLog.md and gets updated after each session. The point of having it here is to remind that:

- Methods aren't tied to single zones. Anchor Ritual has been used for kitchen sink, master bedroom (bed-making), and kids' bathroom — same method, different application, all valid.
- A method can absolutely be reused on a zone where it's appeared before, *if the moment is different*. Three Disaster Chronicles in the kitchen using Triage Rules can all be different posts.
- Empty cells in the matrix are *not* a to-do list. They're just empty cells. Some combinations are obvious (Anchor Ritual + Garage) and don't need filling. Some are weird and shouldn't be forced.

Consult SUSIdeaLog.md for the actual current matrix.

## Reference 6: Reader Stages

The blog historically tagged posts by the kind of reader they served. Useful as a sanity check on whether the blog is overweighted toward one stage.

- **Total Chaos** — drowning, needs immediate triage. Crisis posts, 5-minute façades, panic protocols.
- **System Seeker** — ready to implement routines. Method tutorials, daily defense.
- **Declutter Mode** — tackling the big purge. Zone transformations, category conquests.
- **Maintenance** — has systems, needs to sustain. Anchor rituals, evening lockdowns, command center.
- **Crisis Recovery** — fell off, getting back on. Crisis recovery plans, priority pyramid.

If recent posts skew heavily toward one stage, vary intentionally.

## What This File Is Not

This file is **not** an idea generator. The previous version of it included sample "fingerprints" combining axis values into post concepts — that's gone. The new logic in SUSMaster.md generates ideas from moments, with this file as a consultation resource.

If you find yourself trying to combine axis values to invent a post, stop and go back to "what's the moment?" The moment comes first. Methods, zones, and chaos sources are how you describe a moment after you have one — not a way to generate one.
